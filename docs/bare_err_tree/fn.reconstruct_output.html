Reconstructs ErrTree formatted output from JSON.

Only the output produced by tree_to_json is valid for this function.

FRONT_MAX limits the number of leading bytes. Each deeper error requires 6 bytes to fit “│ “. So for a max depth of 3 errors, FRONT_MAX == 18. By default, FRONT_MAX bytes are allocated on stack. When heap_buffer is enabled, the bytes are allocated on stack and FRONT_MAX only acts as a depth limit. When tracing is enabled, at most FRONT_MAX stack traces will be tracked for duplicates.
